# OpenNotebook 严格对比与融合清单（知识体系优先）

## 对比原则
1. 仅借鉴能提升“知识体系优先级”的能力。
2. 不引入与现有规则/冲突/时效体系冲突的功能。
3. 若能力无法稳定落地，则提供回退方案。

## 功能对比矩阵（节选）

| 能力 | OpenNotebook 常见形态 | 当前项目原始状态 | 融合决策 |
|---|---|---|---|
| 规则编辑工作台 | 独立规则页面、可编辑/删除 | 仅侧栏统计 + API | ✅ 已引入完整规则配置页 |
| 规则可视化 | 表格/分布图/状态视图 | 只有侧栏简版权重条 | ✅ 新增表格+分布图 |
| 规则模拟 | 输入上下文预演命中规则 | 缺失 | ✅ 新增规则模拟器 |
| 外部知识规则导入 | 主题化采集 + 自动抽取 | 仅基础规则接口 | ✅ 新增 web bootstrap + 模板回退 |
| 证据追踪 | 回答附证据/轨迹 | 仅文本回答 | ✅ 新增证据追踪面板 |
| Prompt 模板 | 模板快捷注入 | 缺失 | ✅ 新增模板下拉注入 |

## 融合后仍坚持的“知识优先”约束
- 检索结果与规则系统为第一优先级，模型仅在知识约束内生成。
- 冲突识别、时效规则、来源置信度加权继续生效。
- 外部规则导入失败时，不空转：自动回退到内置规则模板体系。

## 后续可选增强（不影响当前主链路）
- 规则版本化（draft/published）。
- 规则依赖图（scope 与 boundary 关联图谱）。
- 外部来源去重与可信度白名单。
