# 个人知识服务执行偏差复盘与优化闭环（3轮）

## Round 1：复盘执行偏差 → 适配 DOC/PDF 核心链路
### 偏差
- 之前流程偏重 JSON 输入，不符合个人用户“直接上传文档”的真实入口。
- 对文档来源缺少清晰映射规则，难以复现“上传→结构化→可推理”闭环。

### 处理动作
1. 仅支持 `DOC/DOCX/PDF` 上传。
2. 解析规则：
   - PDF：提取文本流字面量（Tj/TJ）
   - DOCX：解包 `word/document.xml` 抽取文本节点
   - DOC：保底二进制文本清洗（可读性 best-effort）
3. 映射规则：`document -> chunk -> KnowledgeUnit(case)`；每块含 `topic/polarity/summary/document/section_index/reinforcement`。
4. 异常机制：
   - 格式不支持：`unsupported_format`
   - base64 非法：`invalid_base64`
   - 内容不可读：`empty_or_unreadable_content`
   - 写入失败：事务回滚（bulk write）

### 验证标准
- 上传后 `documents` 表有记录；
- `knowledge_units` 有对应分块；
- `cognition/run` 可返回决策。

## Round 2：营销客服助手（规章制度检索）价值对比优化
### 角色痛点拆解
- 国家/企业/部门规章层级多，人工查阅耗时高。
- 记忆偏差导致回复不一致。
- 响应窗口短，难在 SLA 内给出可追溯回答。

### 工具介入路径
1. 上传制度 DOC/PDF
2. 自动结构化入知识库
3. 运行认知循环给出约束建议
4. 在 Judgements/Loop Runs 复核解释路径

### 对比指标（示例基线）
- 检索耗时：18.0 min → 4.7 min（+73.9%）
- 首响：140s → 46s（-67.1%）
- 规章错误率：19% → 6%（-68.4%）

### 可执行功能落地
- `/api/documents/upload`
- `/api/cases/marketing-assistant`
- `/api/documents` + 遥测页

## Round 3：9 场景标准化泛化验证
按模板：身份 → 核心工作 → 痛点 → 工具路径 → 价值指标。

1. **慢病患者管理师（健康）**
   - 工作：读取体检/用药说明 PDF，生成日常管理建议
   - 指标：检索时间下降 60%+，遗漏风险项下降 40%+
2. **家庭营养管理师（健康）**
   - 工作：从膳食规范文档生成周计划
   - 指标：方案输出时间减少 50%+
3. **专注力教练（效率）**
   - 工作：复盘行为日志/方法论文档
   - 指标：策略建议首响缩短 45%+
4. **个人理财顾问（金融）**
   - 工作：对照监管与产品说明书
   - 指标：误判率下降 35%+
5. **保险咨询师（金融）**
   - 工作：条款 DOC/PDF 快速比对
   - 指标：条款检索速度提升 65%+
6. **考试备考教练（教育）**
   - 工作：教材/大纲解析→复习任务生成
   - 指标：计划产出时间下降 55%+
7. **职业转型顾问（职业）**
   - 工作：岗位 JD + 能力框架文档匹配
   - 指标：路径建议质量稳定性提升（偏差率降 30%+）
8. **项目复盘教练（职业）**
   - 工作：项目纪要/规范整合
   - 指标：复盘结论一致性提升 40%+
9. **营销客服助手（客服）**
   - 工作：规章制度查询与回复建议
   - 指标：首响与错误率显著下降

### 每场景执行步骤（统一）
1. 上传 DOC/PDF
2. 自动分块结构化
3. 执行 cognition loop
4. 查看决策与遥测
5. 对照基线指标记录提升

### 可复现标准
- 固定接口与字段；
- 固定验证指标；
- 批量数据规模 > 1000/场景；
- 报告落盘 `summary.json + validation_report.md`。
